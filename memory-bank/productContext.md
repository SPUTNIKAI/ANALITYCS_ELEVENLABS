# Product Context

Почему проект существует:
- Автоматизировать прием и сохранение данных из ElevenLabs Agents после звонка/диалога
- Подготовить данные для аналитики разговоров и качества ассистента

Как это должно работать:
- ElevenLabs отправляет POST вебхук на `/webhook/elevenlabs` после завершения разговора
- Сервер валидирует подпись, временное окно, парсит JSON и сохраняет событие
- Возвращает `200 OK` немедленно, чтобы не терять события

Целевые пользователи:
- Разработчики и аналитики, отслеживающие эффективность агента
- Продакшн-операторы, контролирующие стабильность интеграции

UX/операционные ожидания:
- Минимум ручных действий: конфигурации через переменные окружения
- Подробные логи валидации подписи и ошибок
- Опциональный IP-whitelisting источников ElevenLabs

Границы первой итерации:
- Реалтайм-аналитика вне scope; сохраняем события и метаданные `has_*`
