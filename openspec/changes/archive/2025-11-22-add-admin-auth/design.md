## Context
Текущая система не имеет никакой защиты админ-интерфейсов. Это создает риски безопасности, особенно при развертывании в продакшене. Необходимо добавить базовую аутентификацию для защиты доступа к управлению данными.

## Goals / Non-Goals
- Goals:
  - Защитить админ-страницы от несанкционированного доступа
  - Предоставить простой способ аутентификации
  - Использовать минималистичный подход без сложных зависимостей
  - Обеспечить обратную совместимость для API эндпоинтов
- Non-Goals:
  - Создание полноценной системы пользователей
  - Интеграция с внешними провайдерами аутентификации
  - Реализация сложных политик безопасности (2FA, сессии и т.д.)

## Decisions
- **Метод аутентификации**: Простая cookie-based сессия с hardcoded credentials
- **Хранение credentials**: Переменные окружения (ADMIN_USERNAME, ADMIN_PASSWORD)
- **UI аутентификации**: Простая HTML форма без сложного дизайна
- **Защита маршрутов**: Middleware для Express.js
- **Безопасность**: Basic HTTP-only cookie без дополнительных мер

## Risks / Trade-offs
- **Простота vs Безопасность**: Hardcoded credentials уязвимы, но просты в настройке
- **Производительность**: Минимальный overhead на middleware
- **Масштабируемость**: Решение подходит только для одного пользователя
