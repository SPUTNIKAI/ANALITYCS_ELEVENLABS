## Context
Текущий admin.html предоставляет минимальный интерфейс для просмотра событий/лидов в виде простой HTML-таблицы без стилей. Пользователи нуждаются в современном, функциональном интерфейсе для работы с лидами из базы данных.

## Goals / Non-Goals
- Goals:
  - Создать современный веб-интерфейс для просмотра всех лидов
  - Реализовать фильтрацию, поиск и сортировку
  - Обеспечить хорошую производительность при больших объемах данных
  - Сделать интерфейс адаптивным для мобильных устройств
- Non-Goals:
  - Создание сложной системы управления пользователями
  - Интеграция с внешними CRM системами (кроме существующей)
  - Реализация редактирования лидов

## Decisions
- **UI Framework**: Использовать чистый HTML/CSS/JS без тяжелых фреймворков для минимальных зависимостей
- **Стилизация**: Применить современный дизайн с использованием CSS Grid/Flexbox и кастомных стилей
- **API Design**: Расширить существующий `/api/events` endpoint вместо создания нового
- **Пагинация**: Реализовать на стороне сервера для эффективной работы с большими данными
- **Фильтрация**: Добавить параметры запроса для фильтрации по дате, агенту, качеству

## Risks / Trade-offs
- **Производительность**: При большом количестве лидов (>1000) нужна оптимизация запросов
- **Браузерная совместимость**: Чистый JS может иметь ограничения в старых браузерах
- **Мобильная адаптивность**: Требует дополнительного CSS для различных экранов

## Migration Plan
1. Создать новый интерфейс параллельно с существующим
2. Протестировать функциональность на тестовых данных
3. Заменить admin.html на новый интерфейс
4. Обновить документацию и инструкции пользователям
