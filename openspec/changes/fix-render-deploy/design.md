## Context
Render.com имеет проблемы с нашей текущей конфигурацией render.yaml. Несмотря на rootDir: '.', Render все еще ищет package.json в src/ директории.

## Goals / Non-Goals
- Goals:
  - Исправить ошибки деплоя на Render.com
  - Сделать процесс деплоя рабочим
  - Сохранить production-ready конфигурацию
- Non-Goals:
  - Сохранять приватность репозитория (временно)
  - Использовать сложные SSH конфигурации

## Decisions
- **Временно сделать репозиторий публичным** - самый простой способ решить проблему аутентификации
- **Упростить render.yaml** - убрать SSH ключи, которые могут вызывать проблемы
- **Оставить build скрипт** - он работает правильно и нужен для установки зависимостей

## Risks / Trade-offs
- **Временная публичность репозитория** - код будет виден всем, но это временное решение
- **Упрощенная конфигурация** - потеряем SSH безопасность, но получим рабочий деплой

## Migration Plan
1. Сделать репозиторий публичным
2. Упростить render.yaml
3. Протестировать деплой
4. После успешного деплоя вернуть приватность
